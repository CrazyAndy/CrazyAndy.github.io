Spring Cloud 最快上手篇002_服务治理

一、概述

　　服务治理是微服务架构中最核心和基础的模块，因为它的存在才会能实现服务的负载均衡，也就是微服务相比单体应用存在的意义。服务治理主要是分成服务注册和服务发现两块内容。 　　1. 服务注册：我们建立一个服务注册中心，每个服务单元都会向注册中心登记自己提供的服务，将主机与端口号、版本号、通信协议等一些附加信息告知注册中心，注册中心按照服务名为主键分类组织服务清单。另外，服务注册中心还会以心跳的方式去监测清单中的服务是否可用，如果不可用需要从服务清单中剔除。 　　2. 服务发现：由于在服务治理框架下运作，服务间的调用不再通过指定具体的实例地址来实现，而是通过向服务名发起请求调用实现。所以，服务调用方在服务注册中心咨询服务，并获取所有服务的实例清单，以实现对具体服务实例的访问。当服务消费者a调用服务生产者b，而以服务名b注册的服务单元有多个实例在运行，那么a会从服务注册中心下载拉取的清单中以某种轮询策略去出一个位置来进行服务调用，

二、Eureka简介

　　Spring Cloud Eureka 使用Netflix Eureka 来实现服务注册与发现，它包含服务端和客户端组件。 　　1. Eureka服务端，也就是注册中心，它依托于强一致性提供良好的服务实例可用性。 　　2. Eureka客户端，主要处理服务的注册与发现，它向注册中心注册自身提供的服务并周期性地发送心跳来更新它的服务租约。同时，它能从服务端查询当前注册的服务信息并把它们缓存到本地并周期性的刷新服务状态。

三、代码实现

(一)服务注册中心(服务端)

　　1. 建立spring boot工程命名为 eureka-server，并在pom中引入必要依赖 spring-cloud-starter-eureka-server

　　2. 在Application主类上加入注解@EnableEurekaServer

　　3. 在application.properties增加配置，禁用服务注册中心自身的客户端注册行为

    server.port = 1111
    eureka.instance.hostname=localhost
    eureka.client.register-with-eureka=false
    eureka.client.fetch-registry=false
    eureka.client.serviceUrl.defaultZone=http://${eureka.instance.hostname}:${server.port}/eureka

(二)服务提供者(客户端)

　　1. 在主类上通过加上@EnableDiscoveryClient注解

　　2. 在application.properties 需要做相应配置

    spring.application.name=hello-service
    eureka.client.serviceUrl.defaultZone=http://localhost:1111/eureka/
