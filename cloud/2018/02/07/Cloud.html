Spring Cloud 最快上手篇007_分布式配置中心

一、应用场景

　　spring cloud config分为服务端于客户端两个部分，服务端也成微分布式配置中心，他是一个独立的微服务应用，用来连接配置仓库并未客户端提供获取配置信息，加密/解密信息等访问接口；而客户端则是微服务架构中的各个微服务应用或基础设置，它们通过指定的配置中心来管理应用资源与业务相关的配置内容，并在启动的时候从配置中心获取和加载配置信息。它默认采用git来存储配置信息，当然也支持svn仓库，本地化文件系统。

二、快速入门

(一)构建配置中心

　　依赖spring-cloud-config-server，在主类加入注解@EnableConfigServer，在application.properties中添加配置服务的基本信息以及版本管理仓库的相关信息。

(二)客户端配置映射

　　我们需要注意的是我们需要将spring cloud config客户端的配置要放在bootstrap.properties中，只有这样才能从config-server中的配置信息加载到客户端中。
