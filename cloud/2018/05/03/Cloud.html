Spring Cloud问题解决篇003_服务间调用规范

一、问题描述

　　Spring cloud 体系中，为服务间调用是用FeignClient（ribbon 和hystrix的功能合集）的。 　　两个微服务，一个消费者，一个生产者，他们之间通讯最好需要约定一个接口，生产者实现该接口，消费者注入该接口的引用。

二、解决思路

　　现在有user-service 和 ticket-service 两个微服务，它们各自建立独立的 Spring boot工程，现在需要 写一个 user-service 的余额查询服务，ticket-service 需要调用这个服务。下面我们将用代码实现简单逻辑，以下代码完全手敲，可能有拼写错误，请忽略这点不完美。 　　1. 我们首先定义一个通讯的规范标准接口，它需要定义在独立于前两个工程的另外一个单独工程里面，命名为 adapter-api ，其中有一个接口是这么写的 　　

	@RestController
	@RequestMapping(“/adapter”)
	public interface  AdapterApiInf {
		@RequestMapping(“/queryBalanceByUserId”)
		BigDecimal queryBalanceByUserId(String userId);
	}

　　2. 在user-service 中，我们定义实现该接口的controller

	public class AdapterController implements AdapterApiInf {
		@overide 
		public BigDecimal queryBalanceByUserId(String userId){
			return BigDecimal.valueOf(39f); 
		}
	}

　　3. 在ticket-service中，我们用一个service 继承是AdapterApiInf

	@Service
	@FeignClient(name=”USER-SERVICE”)
	public interface AdapterService extends AdapterApiInf {
	}

　　在该项目中的一个controller中，我们这么写

	@RestController
	@RequestMapping(“/ticket”)
	public class TicketController {
		@Autowired
		Private AdapterService  adapterService ;

		@RequestMapping(method=RequestMethod.POST)
		Public String buyTicket(@RequestParam(“userId”) String userId){
			BigDecimal userBalance = adapterService.queryBalanceByUserId(userId);
		}
	}
