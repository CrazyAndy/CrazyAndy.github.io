Spring Cloud问题解决篇008_feign bean not found

一、问题描述

　　feign在配置@FeignClient的时候，如果启动主程序，有可能报错 bean not found。

二、解决思路

　　我们的服务消费者，也就是feign客户端，如果需要调用生产者提供的接口时，代码写法是这样的。

@FeignClient(name = "produce002")
public interface HelloService {
    @RequestMapping("/hello")
    String hello();
}
@RestController
public class ConsumerController {

    @Autowired
    HelloService helloService;

    @RequestMapping("/consume")
    public String consume(){
        String msg = helloService.hello();
        System.out.println(msg);
        return msg;
    }
}

　　在以上代码中，我们的controller注入 helloService的时候，发现找不到它的实现类而报错bean not found， 这说明我们的应用并没有接受 HelloService为feign客户端的形式存在，而是认为其为普通的interface，而spring容器中在为controller注入该service的时候，需要从容器中寻找该service的具体实现类，很显然，实现类是要写在生产者服务中的，所以我们要想办法让该消费者知道HelloService为feignclient，这时候不仅要在HelloService上加注解@FeignClient，而且要在主应用上假如feignclient的扫描注解,代码如下：

@EnableFeignClients(basePackages = "com.demo")//com.demo为HelloService接口所在的包，这样它就能扫描到
@EnableDiscoveryClient
@SpringBootApplication
public class Consumer002Application {

	public static void main(String[] args) {
		SpringApplication.run(Consumer002Application.class, args);
	}
}
