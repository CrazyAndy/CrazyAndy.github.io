Ambari了解一下001_Ambari安装

一、Ambari简介

　　Ambari是Hortonworks开源的Hadoop平台的管理软件，具备Hadoop组件的安装、管理、运维等基本功能，提供Web UI进行可视化的集群管理，简化了大数据平台的安装、使用难度。ambari提供自定义服务功能，我们也可以通过这项功能编写自定义脚本，安装自己想要部署的服务应用。

二、安装环境要求

（一）集群节点说明

　　我们将会拿出ip为 172和118两台机器做测试，172安装server端和agent端，118只是安装agent端。

192.168.100.172  devserver06（ambari主节点）
192.168.100.118  devserver08 （agent节点）

（二）安装包准备

ambari-2.6.0.0-centos7.tar.gz
HDP-UTILS-1.1.0.21-centos7.tar.gz
HDP-2.6.3.0-centos7-rpm.tar.gz
jdk-8u112-linux-x64.tar.gz//如果当前机器已经安装jdk,请确保版本在1.8以上即可
jce_policy-8.zip
mysql-connector-java.jar//我们使用mysql作为ambari的配置数据库，当然如果使用其他数据库则也可以将此替换成其他驱动包
ambari.repo
hdp.repo

三、设置ssh免密登陆

（一）生成ssh公钥私钥

　　root 用户登陆下，执行命令：ssh-keygen -t rsa 执行完毕后，在/root/.ssh/ 目录下面会有 id_rsa id_rsa.pub这两个文件

（二）修改/etc/profile

　　我们需要让机器互认，就需要让他们知道对方的机器名称，可以在/etc/hosts里面声明 ambari-server端，需要记录本机机器名称(可以通过hostname命令查看)，还有各个ambari-agent节点的机器名称，记录内容举例如下所示

192.168.100.172  devserver06
192.168.100.118  devserver08

（三）推送公钥到ambari agent节点

　　ssh-copy-id -i id_rsa.pub root@devserver08

（四）测试ssh登陆

　　通过命令 ssh devserver08 进行测试，第一次测试，会需要你确认，以后不用再确认

（五）备注

　　如果在进行以上四部操作之后还不能免密登陆，那么可以尝试修改 .ssh文件夹以及其目录下文件的操作权限，都改成700，命令为：chmod 700 /root/.ssh

四、安装ntp服务

　　安装ambari集群需要保证集群内机器时间同步，我们需要在所有机器上都安装ntp服务，安装命令如下：

yum install -y ntp
chkconfig ntpd on

五、防火墙设置

service firewalld stop 
systemctl disable firewalld

六、安装httpd服务

　　由于安装ambari需要拉取安装文件，但是文件有6G之多，拉取时间超过半小时，ambari默认安装失败并退出安装，所以我们需要在主机节点搭建httpd服务，而且安装自定义服务的时候，我们可以把需要的程序包放到此服务下面。

（一）安装httpd

　　首先查看 httpd服务是否已经存在，systemctl status httpd  安装命令如下：# yum install httpd 　　我们可以通过修改httpd.conf文件来修改httpd服务配置(端口，文件存放位置等) 配置文件默认在 /etc/httpd/conf/httpd.conf 　　文件存放目录默认为/var/www/html，如果不存在，需要手动创建

（二）上传ambari安装包

　　我们将以下文件上传到 /var/www/html目录下

ambari-2.6.0.0-centos7.tar.gz
HDP-UTILS-1.1.0.21-centos7.tar.gz
HDP-2.6.3.0-centos7-rpm.tar.gz

　　并在/var/www/html目录下面进行解压

tar -zxf ambari-2.6.0.0-centos7.tar.gz -C /var/www/html/ 
tar -zxf HDP-2.6.3.0-centos7-rpm.tar.gz -C / var/www/html/hdp 
tar -zxf HDP-UTILS-1.1.0.21-centos7.tar.gz -C / var/www/html/hdp

（三）确认httpd服务是否能正常使用

　　通过访问http://192.168.100.172/hdp来验证

七、安装ambari-server

（一）配置repo源

　　我们需要通过yum命令安装ambari，所以需要修改repo文件，让其指向之前搭建的http服务地址(一定要确认真正能访问到) 　　修改之前准备好的ambari.repo和hdp.repo文件内容，将内容中url指向正确能用的地址，并上传这两个文件到集群各个节点 /etc/yum.repos.d/目录下

（二）安装ambari-server

　　在192.168.100.172主节点上我们安装ambari-server端，其他节点安装ambari-agent端 yum install ambari-server 操作中会提示 is this ok 请输入y

（三）配置ambari数据库

　　/var/lib/ambari-server/resources下有数据库执行脚本文件Ambari-DDL-MySQL-CREATE.sql Ambari运行需要有自己的数据库，我们使用mysql数据库，我们需要将Ambari-DDL-MySQL-CREATE.sql文件在数据库机器上执行，安装对应数据库，并创建数据库用户，而且还要允许该用户在主机节点可以访问该数据库。

（四）配置ambari-server

　　我们需要将提前准备好的jce_policy-8.zip，jdk-8u112-linux-x64.tar.gz这两个文件上传到主节点 c目录下 server.jdbc.driver.path=[path/to/custom_jdbc_driver] 　　我们需要将mysql-connector-java.jar 上传到/usr/share/java上去，这样ambari才能正常连接mysql数据库 执行命令ambari-server setup

　　数据库访问用户密码需要在数据库端创建用户，并设定密码(密码不能为空)，在以上Ambari-server setup的时候，需要填入用户密码

　　Ambari的配置文件位置为 /etc/ambari-server/conf/ambari.properties安装成功之后,需要启动ambari-server，执行命令为ambari-server start

九、安装ambari-agent

（一）安装

　　我们需要在ambari集群下的所有agent节点安装此命令 　　yum install ambari-agent即可

（二）修改/etc/hosts

　　我们需要让agent能认识主节点机器 　　Vi /etc/hosts 　　192.168.100.172 devserver06

（三）修改agent配置

　　vi /etc/ambari-agent/conf/ambari-agent.ini 　　修改 server hostname，内容如下

[server]
hostname=devserver06

（四）在ambari-server上增加agent host

　　点击 Add New Hosts ，界面如下
