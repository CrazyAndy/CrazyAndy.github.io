docker-7-私服

Registry2.0

docker 私服

我先使用的是registry2.0，这个界面也没有，以后准备换成其他的

安装私服

docker pull registry

本地添加私服

我们需要创建/etc/docker/daemon.json文件

内容如下

{
  "registry-mirrors": [
    "https://registry.docker-cn.com"
  ],
  "insecure-registries": [
    "192.168.111.133:5000"
  ]
}

然后重新docker

service docker restart

查看是否配置成功

docker info

上传下载镜像

由于没有配置ssl，我们不得不使用 ip:port/name:tag 来上传下载镜像

Harbor

介绍

我们推荐Harbor来取代Registry2.0的原因是，harbor自带管理系统页面，能很直观的看到你所有的仓库内容。甚至还有用户管理等内容。 安装harbor也很简单，主要是下载tgz压缩包，修改配置文件，编译运行。

https://www.cnblogs.com/guyeshanrenshiwoshifu/p/9166195.html

https://blog.csdn.net/qq_43309149/article/details/88608726

还是做下笔记吧

1，下载目录

https://github.com/goharbor/harbor/releases

2，修改harbor.yml 配置文件，修改域名，mysql和redis 数据库（可以改成自己的）

3,运行 prepare 执行脚本

这时候可能会提示

expected , but found '' in "/input/harbor.yml", line 102, column 2

之类的问题，主要是配置文件 ，要：后面有一个空格，而且 层级结构要对。

4，执行 ./install.sh 脚本

需要docker-compose 版本在 1.18.0以上

5，升级docker-compse

https://docs.docker.com/compose/install/

6,修改 harbor.yml

修改域名，端口号，外置数据库，关系型职能只支持 POSTGRES，可以配置redis，但是必须用0的索引，恶心。
