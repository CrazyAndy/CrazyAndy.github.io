OAuth2.0探索(三)java代码实战

一、概述

　　我们需要开发oauth-client 和 oauth-server两个web项目，

　　oauth-client负责开发第三方应用程序，也就是客户端程序，如果对应第一章就是云冲印，如果对应第二章就是豆瓣。

　　oauth-server负责开发服务提供商，主要包括认证服务器和资源服务器两部分。对应第一章就是谷歌，第二张就是qq。

　　在实际业务流程中，http请求会从这两个web服务来回跳转，可有意思了^_^。

（二） 用户点击客户端页面，确认授权

　　上面的点击确认授权，实质上是请求 http://localhost:8091/oauthclient01/requestServerCode对应时序图中的1阶段

　　oauthclient01/requestServerCode这个请求对应的后台代码在oauth-client项目ClientServerController类中，对应时序图中的2阶段，代码如下所示

@Controller
public class ClientServerController {
    String clientId = null;
    String clientSecret = null;
    String accessTokenUrl = null;
    String userInfoUrl = null;
    String redirectUrl = null;
    String response_type = null;
    String code= null;

    @Autowired
    private HttpServletRequest request;
    @Autowired
    private HttpServletResponse response;

    @RequestMapping("/requestServerCode")
    public String requestServerFirst(RedirectAttributes attr) throws OAuthProblemException {
        clientId = "clientId";
        clientSecret = "clientSecret";
        accessTokenUrl = "responseCode";
        redirectUrl = "http://localhost:8091/oauthclient01/callbackCode";
        response_type = "code";
        OAuthClient oAuthClient =new OAuthClient(new URLConnectionClient());
        String requestUrl = null;

        try {
            //构建oauth的请求。设置请求服务地址（accessTokenUrl）、clientId、response_type、redirectUrl
            OAuthClientRequest accessTokenRequest = OAuthClientRequest
                    .authorizationLocation(accessTokenUrl)
                    .setResponseType(response_type)
                    .setClientId(clientId)
                    .setRedirectURI(redirectUrl)
                    .buildQueryMessage();
            requestUrl = accessTokenRequest.getLocationUri();
            System.out.println(requestUrl);
        } catch (OAuthSystemException e) {
            e.printStackTrace();
        }
        return "redirect:http://localhost:8082/oauthserver/"+requestUrl ;
    }
}
