OAuth2.0探索(三)java代码实战

一、概述

　　我们需要开发oauth-client 和 oauth-server两个web项目，

　　oauth-client负责开发第三方应用程序，也就是客户端程序，如果对应第一章就是云冲印，如果对应第二章就是豆瓣。

　　oauth-server负责开发服务提供商，主要包括认证服务器和资源服务器两部分。对应第一章就是谷歌，第二张就是qq。

　　在实际业务流程中，http请求会从这两个web服务来回跳转，可有意思了^_^。

二、java实战描述

（一） 序列图

（四）接受服务端端返回的code，提交申请access token的请求

　　代码位于oauth-client中类ClientServerController中

    //接受服务端端返回的code，提交申请access token的请求
    @RequestMapping("/callbackCode")
    public Object callbackCode()throws OAuthProblemException{

        System.out.println("-----------客户端/callbackCode--------------------------------------------------------------------------------");
        clientId = "clientId";
        clientSecret = "clientSecret";
        accessTokenUrl="http://localhost:8082/oauthserver/responseAccessToken";
        userInfoUrl = "userInfoUrl";
        redirectUrl = "http://localhost:8091/oauthclient01/accessToken";

        HttpServletRequest httpRequest = (HttpServletRequest)request;
        code = httpRequest.getParameter("code");
        System.out.println(code);
        OAuthClient oAuthClient =new OAuthClient(new URLConnectionClient());
        try {
            OAuthClientRequest accessTokenRequest = OAuthClientRequest
                    .tokenLocation(accessTokenUrl)
                    .setGrantType(GrantType.AUTHORIZATION_CODE)
                    .setClientId(clientId)
                    .setClientSecret(clientSecret)
                    .setCode(code)
                    .setRedirectURI(redirectUrl)
                    .buildQueryMessage();

            //去服务端请求access token，并返回响应
            OAuthAccessTokenResponse oAuthResponse =oAuthClient.accessToken(accessTokenRequest, OAuth.HttpMethod.POST);

            //获取服务端返回过来的access token
            String accessToken = oAuthResponse.getAccessToken();

            //查看access token是否过期
            Long expiresIn =oAuthResponse.getExpiresIn();
            System.out.println("过期时间："+expiresIn);
            System.out.println("客户端/callbackCode方法的token：：："+accessToken);
            System.out.println("-----------客户端/callbackCode--------------------------------------------------------------------------------");
            return "redirect:http://localhost:8091/oauthclient01/accessToken?accessToken="+accessToken;
        } catch (OAuthSystemException e) {
            e.printStackTrace();
        }
        return null;
    }

（五） 获取客户端的code码，向客户端返回access token

　　在第四步中的代码中，客户端会自动请求服务端，用code换取token，而服务端代码放在oauth-server中AccessTokenController类中

@RestController
public class AccessTokenController {

    @Autowired
    private HttpServletRequest request;
    @Autowired
    private HttpServletResponse response;

    //获取客户端的code码，向客户端返回access token
    @RequestMapping(value="/responseAccessToken",method = RequestMethod.POST)
    public HttpEntity token(){

        System.out.println("--------服务端/responseAccessToken-----------------------------------------------------------");

        OAuthIssuer oauthIssuerImpl=null;
        OAuthResponse response=null;

        //构建OAuth请求
        try {
            OAuthTokenRequest oauthRequest =new OAuthTokenRequest(request);
            String authCode =oauthRequest.getParam(OAuth.OAUTH_CODE);
            String clientSecret = oauthRequest.getClientSecret();
            if(clientSecret!=null||clientSecret!=""){
                //生成Access Token
                oauthIssuerImpl =new OAuthIssuerImpl(new MD5Generator());
                final String accessToken =oauthIssuerImpl.accessToken();

                System.out.println(accessToken);
                System.out.println("--oooo---");
                //生成OAuth响应
                response = OAuthASResponse
                        .tokenResponse(HttpServletResponse.SC_OK)
                        .setAccessToken(accessToken)
                        .buildJSONMessage();
            }
            System.out.println("--------服务端/responseAccessToken-----------------------------------------------------------");
            //根据OAuthResponse生成ResponseEntity
            return new ResponseEntity(response.getBody(), HttpStatus.valueOf(response.getResponseStatus()));
        } catch (OAuthSystemException e) {
            e.printStackTrace();
        } catch (OAuthProblemException e) {
            e.printStackTrace();
        }

        System.out.println("--------服务端/responseAccessToken-----------------------------------------------------------");
        return null;
    }
}

　　我们在第六步代码最后将第七步返回内容显示在页面上，至此，我们完成了oauth2.0整套流程逻辑，当然在服务器端AccessTokenController中token()方法中，我们并没有真正将code换token，这个需要clientid,code,token它们形成的一条记录要事先存储到数据库或其他地方的，我们并没有做深入的实现，只是在那搭了个架子而已。

　　而实际生产中，我们一般是需要页面重定向到认证服务器的时候，是要有界面提供方便用户输入账号密码的，然后才能得到code值，而这我们也没有做详细实现，在此要说明以下，以免读者的你误会或者总感觉哪里缺少什么却说不出来。
