使用jekyll搭建github博客

零、创建个人博客的好处

　　当程序猿就是一个入坑，爬坑，再入坑的过程，循环往复，无休无止。毫不谦虚的说，我们程序猿就是战士。

　　写程序写了这几年，也看了一些书，看了别人一些代码，和他人讨论过软件设计，当然，也经历过几次面试，大言不惭的说，失败的居多。可是心凉之后，我们要想想，为什么我们的输出并不像我们平时自认为的那样，人家问你你都含含糊糊，虽然可能用过，却又不能说出个123来。为什么呢？我觉得就是基本功不扎实，用功要在一朝一夕，切不可急于求成，临时抱佛脚，你即便应付得了笔试题，却会在老师傅面前分分钟被打回原形。

　　至少我很多次在学习新技术的时候发现，看一本书很容易，一周的业余时间就能翻完，可是如果你将书中的只是应用于时间，那你这本书却要在之后的时间里翻个很多遍。写博客介绍该技术也是，你需要对这门技术各个方面都要有个比较清晰的认知，才能写的出来，也就是才会有好的“输出”。而当你产生好的输出的时候，你的能力也就跟上来了。

　　好多技术书籍的作者都是在某一技术领域写心得写多了，然后改吧改吧就编册成书了，这是真哒!

一、jekyll描述

　　Jekyll 是一个简单的博客形态的静态站点生产机器。它有一个模版目录，其中包含原始文本格式的文档，通过 Markdown （或者 Textile） 以及 Liquid 转化成一个完整的可发布的静态网站，你可以发布在任何你喜爱的服务器上。Jekyll 也可以运行在 GitHub Page 上，也就是说，你可以使用 GitHub 的服务来搭建你的项目页面、博客或者网站，而且是完全免费的。

二、jekyll环境配置

　　1. 安装ruby环境

　　windows安装ruby环境特别简单，新版本的ruby安装包已经自带devkit,这样省的安装单纯ruby之后再找对应版本devkit，下载网址为https://rubyinstaller.org/downloads/,选择with devkit 下载安装即可。 　　2. 切换gem source源

　　使用gem国外的源安装jekyll特别慢，我们需要切换源，阿里巴巴已经停止维护gem 源,推荐使用ruby china源。但是你可以访问阿里巴巴源，参照其提供的安装方式，地址为 https://ruby.taobao.org/

　　3. 安装jekyll

　　安装jekyll前先按照依赖包bundler，下述命令即可安装 　　 gem install bundler 　　 　　之后，直接可以通过下述命令安装jekyll 　　 gem install jekyll 　　 　　使用 jekyll -v 查看版本，如果有版本出现则证明安装成功

三、jekyll目录结构介绍

　　正常的Jekyll工程包含以下几个目录：

　　　　_posts  博客内容
        _pages  其他需要生成的网页，如About页
        _layouts 网页排版模板
        _includes 被模板包含的HTML片段，可在_config.yml中修改位置
        assets 辅助资源 css布局 js脚本 图片等
        _data 动态数据
        _sites  最终生成的静态网页

　　在项目文件夹(含有_config.yml)中运行jekyll build 指令后，jekyll会依次做如下几件事

　　加载_layout文件夹内的所有模板，并将其中的{ % include xxxx.html % }字段(真实使用中{ 和 % 之间没有空格，这里空出一格，是因为jekyll编辑此博客的时候会认为该博客要引入xxx.html，显然我的博客代码目录中是没有xxx.html的)按照_includes文件夹内对应文件填入遍历_post文件夹及子文件夹，对所有命名符合yyyy-mm–dd-title.md 格式的博客文件放入site.posts 变量(按时间倒序)，并对其进行解析，根据Front Matter 头的内容套入layout生成对应title的博客 遍历整个项目子目录，扫描所有含Front Matter 头的文件，放入site.pages 变量并根据permalink 字段指定的URL目标位置生成index.html

四、jekyll模板下载

　　可以从网上搜一些jekyll模板拿来修改，没必要重新搭建，http://jekyllthemes.org/就是一个不错的jekyll模板网站，我首页的内容就是参照了其中的一个模板。当然博客主体架构还是参照的https://github.com/jarrekk/Jalpc，在此对他们的无私奉献做出感谢。

五、jekyll根据模板创建个人博客

　　其实，你可以在本地自己创建一个文件夹，直接在里面执行以下脚本即可，但是页面是一片荒凉，还是用人家现成的吧。 jekyll new 你的博客根目录文件夹名称

　　在下载了模板之后，你只需要在_posts文件夹下增加一片新的后缀名为.md的文件即可，文件内容参考该模板里_posts其他文件

六、jekyll插件

　　jekyll中文官网是https://www.jekyll.com.cn/，里面有jekyll使用文档，可以随时查看。

　　jekyll支持分页，需要使用gem安装jekyll-paginate插件，具体命令如下

	gem install jekyll-paginate

七、使用markdown编写博客

　　我们需要使用markdown编写后缀名为.md的文件，并把文件放入到_post文件夹下某一级目录下。markdown使用的是liquid语法，它能被jekyll编译成静态html，关于语法细则请自行百度。markdown软件我推荐使用haroopad，而不是使用markdown pad。haroopad相比来说更好用一点。

八、上传jekyll到github博客

　　我们根据模板修改成自己满意的网页的时候，便可将_site下的文件上传到github上去，github在此操作之前需要创建自己的github pages 仓库，具体方法可自行百度。这里不再赘述。
